Get All Tickets

View the user's pending and closed tickets. Requires authentication.

  • Command

    GET /tickets

  • Sample Request

    GET https://api.badger.com/tickets

  • Sample Response

    { "pending_tickets": [ { "id": 1, "created_at": "2012-01-28T07:39:29Z", "updated_at": "2012-01-31T07:39:29Z", "status": "open", "category": "Request Feature", "subject": "Feature Request", "content": "I want to request a new feature", "person": { "name": "John Doe", "id": 1 } }, { "id": 2, "created_at": "2011-11-28T07:39:29Z", "updated_at": "2012-01-14T07:39:29Z", "status": "waiting for user response", "category": "Request Feature", "subject": "New Tool", "content": "I want to request a new feature", "person": { "name": "John Doe", "id": 1 } } ], "closed_tickets": [ { "id": 3, "created_at": "2011-12-28T07:39:29Z", "updated_at": "2012-01-9T07:39:29Z", "status": "closed", "category": "Billing Inquiry", "subject": "Problem with credits", "content": "I have problem with purchasing credits", "person": { "name": "John Doe", "id": 1 } } ] }

Get Ticket Information

Get detailed information of a ticket. Requires authentication.

  • Command

    GET /tickets/:id

  • Parameters (required values are in bold)

    Name Description
    id ID of the ticket

  • Sample Request

    GET https://api.badger.com/tickets/1

  • Sample Response

    { "id": 1, "created_at": "2012-01-28T07:39:29Z", "updated_at": "2012-01-31T07:39:29Z", "status": "open", "category": "Request Feature", "subject": "Feature Request", "content": "I want to request a new feature", "person": { "name": "John Doe", "id": 1 } }, "responses": [{ "id": 2, "ticket_id": 1, "person": { "name": "Admin", "id": 3 }, "response": "Can you give me more details" } ] }

Submit a Support Ticket

Submit a Support Ticket. Requires authentication.

  • Command

    POST /tickets

  • Parameters (required values are in bold)

    Name Description
    category Category of the ticket
    subject Subject of the ticket
    content Content of the ticket

  • Sample Request

    POST https://api.badger.com/tickets?subject=Request&category=Others&content=content

  • Sample Response

    { "message": "Ticket created" }

Reply To A Ticket

Submit a response to a pending ticket. Requires authentication.

  • Command

    POST /tickets/:id/add_response

  • Parameters (required values are in bold)

    Name Description
    id ID of the ticket
    response Response message to the ticket

  • Sample Request

    POST https://api.badger.com/tickets/1/add_response?response=response

  • Sample Response

    { "message": "Response added" }
